一、从用户视角：她为什么会变成“一句我一句”聊天

在你眼里，大概是这样一轮一轮发生的：

你在输入框打字 → 点“发送”。

你的那句话立刻出现在右边（自己的气泡）。

输入框变灰 / 发送按钮不可用，顶部显示“她正在思考…”。

过一小会儿，左边先出现一个灰色“正在回复”的占位，然后她开始一段一段地把话“打出来”。

等她这轮说完，输入框重新可以输入，你再说下一句……

多轮之后，她会记住你说过的一些重要事，再用在后面的对话里。

这个节奏就是“你一句 → 她一句 → 你再一句”，看起来很自然。

二、核心思想：每个女友有一份“聊天记录数组”，每发一条就往后 append

在代码里，每个 AI 女友对应一个 Chat 对象，其中有一条消息数组：

type Message = {
  id: string;
  role: 'user' | 'assistant' | 'system';
  content: string;
  imageUrl?: string;
  timestamp: number;
};

type Chat = {
  personalityId: string;     // 哪个女友
  messages: Message[];       // 跟她的整个聊天记录
  ...
};


在 App.tsx 里，用一个 chats 的 map 存所有女友的聊天：

const [chats, setChats] = useState<Record<string, Chat>>({});
// key 基本就是 personalityId


当你在列表页点某个女友时：

currentPersonalityId 被设置成那个人；

App 找出 chats[currentPersonalityId] 里的 messages；

把这条消息数组通过 props 传给 ChatInterface，显示为聊天窗口。

所以：你看到的聊天就是 messages[] 这条数组的可视化，一条一条按时间追加。

三、发送消息这一瞬间：先插入“我的气泡”，再锁输入框

ChatInterface.tsx 里有一个 sendMessage 函数，你每次点击发送 / 按 Enter 就触发它。关键步骤是：

1. 先做校验 + 创建一条用户消息
if (!input.trim() && !selectedImage) {
  toast.error('请输入消息或选择图片');
  return;
}

const userMessage: Message = {
  id: Date.now().toString(),
  role: 'user',
  content: input.trim() || '[图片]',
  imageUrl: selectedImage || undefined,
  timestamp: Date.now(),
};


没打字又没选图片 → 直接提示。

用当前文本 & 图片构造一条 user 消息。

2. 把这条用户消息直接 append 进当前对话的 messages
const newMessages = [...messages, userMessage];

setMessages(newMessages, true); // true = 清空未读数
setInput('');
setSelectedImage(null);
if (fileInputRef.current) fileInputRef.current.value = '';
setIsLoading(true);


setMessages 是从 App 传进来的，内部会更新 chats[currentPersonalityId].messages。

UI 立刻多出一个自己的气泡；

输入框清空、图片清空；

isLoading = true 表示“这轮我已经说完，正在等她回答”。

3. 在“等待期间”禁止你再来一句

在输入框和发送按钮上都用了 isLoading：

<textarea
  disabled={isLoading || isStreaming}
  ...
/>

<Button
  disabled={isLoading || !input.trim()}
>
  发送
</Button>

const handleKeyDown = (e: React.KeyboardEvent<HTMLTextAreaElement>) => {
  if (e.key === 'Enter' && !e.shiftKey && !isLoading) {
    e.preventDefault();
    sendMessage();
  }
};


isLoading === true 时：

输入框 disabled；

发送按钮 disabled；

Enter 也不会触发发送。

这就保证了：一轮里你只能说一句。必须等她回复完 setIsLoading(false) 之后，才能开始下一轮。

四、AI 回复：把“对话历史 + 人设 + 记忆 + RAG + 联网”扔给 Gemini

接下来就是 AI 女友思考的环节，代码在 sendMessage 的 try { ... } 里：

1. 拼出完整的“上下文”

这里有三部分来源：

你刚刚这句话和之前的聊天记录

newMessages 就是 [...旧消息, 新的 userMessage]。

发给模型时会转成：

const apiMessages = [
  { role: 'system', content: buildSystemPrompt() },
  ...newMessages.map(msg => ({
    role: msg.role,
    content: msg.content
  }))
];


女友的人设（system prompt）

函数 buildSystemPrompt() 会把：

角色名字、背景、性格、经历、喜好

说话风格（撒娇、傲娇、学姐、姐姐…）

以及一些“不要暴露自己是 AI”的规则
都拼成一段长的 system prompt。

记忆 + RAG + 联网（视配置而定）

如果开了 RAG：

用 embedding 提取对当前问题最相关的历史片段 / 知识库文档，拼进 prompt。

如果开了联网搜索：

在 Gemini 那侧打开 googleSearch 工具，让模型在回答前做 web 检索。

这些逻辑都封装在 generateWithRAGAndSearch 里。

2. 按情况选用不同的封装函数

有图片 → generateWithVision

开了 RAG 或联网 → generateWithRAGAndSearch

普通对话 → generateText

以普通对话举例：

const result = await generateText(
  apiMessages,
  config.geminiApiKey,
  actualModel
);

assistantMessage = {
  id: (Date.now() + 1).toString(),
  role: 'assistant',
  content: result.text,
  timestamp: Date.now(),
};


generateText 内部负责：

按 Google API 要求把 apiMessages 转成 contents；

带上 temperature / topP / maxOutputTokens 等生成参数；

调用 https://generativelanguage.googleapis.com/v1beta/models/${model}:generateContent；

从返回的 candidates 里抽出最终文本。

这一整步就是“她思考 + 生成这一句回复”。

五、她怎么变成“分段说话”的？

你在项目里做了一个小特效：用 \ 把回复切成几段，模拟她连续发多条消息。

const segments = assistantMessage.content.split('\\').filter(s => s.trim());

if (segments.length > 1) {
  setIsStreaming(true);
  setMessages(newMessages); // 先只保留到你那一句

  for (let i = 0; i < segments.length; i++) {
    await new Promise(resolve => setTimeout(resolve, 2000 + Math.random() * 1000));

    const segmentMessage: Message = {
      id: `${assistantMessage.id}-seg-${i}`,
      role: 'assistant',
      content: segments[i].trim(),
      timestamp: Date.now(),
    };

    setMessages(prev => [...prev, segmentMessage]);
  }

  setIsStreaming(false);
} else {
  setMessages([...newMessages, assistantMessage]);
}


如果 AI 的 content 里包含多个 \ 分隔的段落：

比如 "第一段...\第二段...\第三段..."

就会：

先什么也不追加，只保留你那条最新的 user 消息；

然后每隔 2～3 秒 append 一条新的 assistant 消息；

直到所有段落发完，isStreaming = false。

这样看起来就像：

她不是一次把一大段扔出来，而是“连续给你发了好几条”。

六、这一轮结束：解锁输入框，允许进入下一轮

无论是哪种调用方式，只要走出 try / catch，最后都会执行：

finally {
  setIsLoading(false);
}


这会重新把 isLoading 置成 false；

输入框和发送按钮的 disabled 解除；

你就可以输入下一句话了。

所以完整的回合是：

你发一条 → isLoading = true → 禁止你再发；

她算完、分段输出 → 消息数组追加几条 assistant；

结束 → isLoading = false → 你再次获得说话权。

这就是“一句我一句”聊天的循环机制。

七、额外加成：她会记住你说过的重要话

每当对话长度到一定数量（比如每 5 条），你还会触发一次“记忆抽取”：

const finalMessages = [...newMessages, assistantMessage];

if (config.geminiApiKey && finalMessages.length % 5 === 0 && finalMessages.length >= 5) {
  const memoryResult = await extractMemoriesFromConversation(
    finalMessages,
    config.geminiApiKey,
    actualModel
  );
  // 把 memoryResult.memories 整理成 Memory[], 更新到 personality.memories
}


extractMemoriesFromConversation 用 Gemini 再读一遍最近对话，让模型挑出可以记住的事实（比如你喜欢什么、最近在做什么）。

这些记忆写到 personality.memories 里，后面在 RAG 和 prompt 里会带上。

所以你不是在“单纯一问一答”，而是在建立一段会成长的关系——每一轮一来一回，都会有机会变成她的记忆。
