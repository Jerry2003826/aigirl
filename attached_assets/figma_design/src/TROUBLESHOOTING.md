## 问题0: 登出后再次登录无限加载 🔄

### 症状
- 登出后再次登录（邮箱或OAuth）
- 页面一直显示加载状态
- 无法进入聊天界面

### 原因
这是一个已知并已修复的bug，如果你遇到这个问题，说明使用的是旧版本代码。

### 解决方案

#### ✅ 已在最新版本修复
最新代码已经完全修复此问题，包括：
- 登出时完整清理所有状态（loading、认证、数据）
- 数据加载失败时正确设置loading状态
- 确保每次登录都是干净的初始状态

#### 🚨 如果仍然遇到此问题
1. **刷新页面** - 按F5或Cmd+R完全刷新
2. **清除浏览器缓存和Cookie**
   - Chrome: 设置 → 隐私和安全 → 清除浏览数据
   - 选择"Cookie和其他网站数据"+"缓存的图片和文件"
3. **重启浏览器**
4. **检查控制台** - 打开开发者工具(F12)查看是否有错误信息

#### 🧪 验证修复
正确的登录流程应该是：
1. 点击登出 → 看到"已登出"提示
2. 再次登录 → 看到"正在完成登录..."提示
3. 1-3秒后 → 进入聊天界面
4. 没有无限加载状态

## 问题1: "发送失败，请检查API配置" ❌

### 症状
- 发送消息时提示"发送失败"
- 错误信息提到"请检查API配置"
- AI无法回复消息

### 原因
**每个用户的API配置是独立存储的。** 即使其他人的API Key有效，新登录的用户也需要单独配置自己的API Key。

### 解决方案

#### 方法1: 通过弹窗提示配置（推荐）⭐
1. 登录后等待5秒
2. 系统会自动弹出"需要配置API Key"提示
3. 点击"打开设置配置"按钮
4. 在"AI配置"中粘贴你的Gemini API Key
5. 点击"保存配置"

#### 方法2: 手动打开设置
1. 点击右上角"⚙️设置"按钮
2. 在"AI配置"卡片中找到"Gemini API Key"输入框
3. 粘贴你的API Key（以`AIza`开头）
4. 点击"保存配置"

#### 方法3: 获取新的API Key
如果没有API Key：
1. 访问 [Google AI Studio](https://aistudio.google.com/apikey)
2. 使用Google账号登录
3. 点击"Create API Key"
4. 复制生成的API Key
5. 返回应用，按方法1或方法2配置

### 验证配置成功
- 配置保存后，尝试发送一条消息
- 如果AI正常回复，说明配置成功 ✅
- 配置会自动保存到云端，其他设备登录后会自动同步

## 问题3: API Key已配置，但仍然失败 ❌

### 症状
- 已经配置了API Key
- 发送消息仍然失败
- 错误信息提到"API错误"或"API key"

### 可能原因和解决方案

#### 原因1: API Key无效
**检查方法**:
1. 进入设置 → AI配置
2. 确认API Key以`AIza`开头
3. 确认没有多余的空格或换行

**解决方案**:
- 重新复制API Key（不要手动输入）
- 在Google AI Studio重新生成一个新的API Key

#### 原因2: API配额用完
**检查方法**:
- 查看错误信息是否包含"quota"或"配额"
- 访问 [Google AI Studio](https://aistudio.google.com/) 查看使用量

**解决方案**:
- 等待配额重置（通常每天或每月）
- 升级到付费计划
- 创建新的Google账号和API Key

#### 原因3: 网络问题
**检查方法**:
- 打开浏览器控制台（F12）
- 查看是否有网络错误（红色的fetch failed等）

**解决方案**:
- 检查网络连接
- 尝试关闭VPN/代理
- 换一个网络环境（移动热点等）

#### 原因4: API Key地区限制
某些地区可能无法直接访问Gemini API。

**解决方案**:
- 使用代理或VPN
- 选择支持的地区

## 问题3: 登录后数据丢失 📦

### 症状
- 登录后看不到之前创建的AI角色
- 聊天记录消失
- 配置被重置

### 原因
可能使用了不同的登录方式。**社交登录和邮箱登录是独立的账号。**

### 解决方案

#### 确认登录方式
1. 检查你之前是如何登录的：
   - 邮箱注册/登录
   - Google账号
   - Microsoft账号
2. 使用**相同的**登录方式重新登录

#### 如果忘记了登录方式
- 查看登录后显示的邮箱地址
- Google/Microsoft登录会显示对应的邮箱
- 邮箱注册会显示注册时使用的邮箱

#### 数据恢复
如果确实登录了正确的账号但数据丢失：
1. 点击左侧菜单
2. 选择"数据恢复"
3. 使用恢复工具检查和恢复数据

## 问题4: 群聊中AI不回复 🤖

### 症状
- 创建了群聊
- 添加了AI成员
- 发送消息后AI没有回复

### 可能原因和解决方案

#### 原因1: 未配置API Key
**解决方案**: 参考问题1配置API Key

#### 原因2: AI在冷却中
群聊有防刷机制，AI回复后需要冷却2秒。

**解决方案**: 等待几秒后再次发送

#### 原因3: 消息内容与AI不匹配
系统会自动选择最相关的AI回复。

**解决方案**: 
- 使用`@AI名字`指定特定AI回复
- 例如：`@小雪 你觉得呢？`

#### 原因4: 话题已达到上限
每个话题默认最多10轮对话。

**解决方案**:
- 点击群聊右上角设置
- 点击"重置话题"
- 或者在设置中增加话题最大轮数

## 问题5: 头像上传失败 🖼️

### 症状
- 点击"上传头像"后没反应
- 上传后头像不显示
- 提示上传失败

### 可能原因和解决方案

#### 原因1: 图片太大
**解决方案**:
- 确保图片小于5MB
- 使用在线工具压缩图片
- 推荐大小：500KB以内

#### 原因2: 图片格式不支持
**解决方案**:
- 只支持JPG和PNG格式
- GIF等动图暂不支持
- 转换图片格式后重试

#### 原因3: 网络问题
**解决方案**:
- 检查网络连接
- 重试上传
- 刷新页面后再试

#### 原因4: 存储空间问题
Supabase免费版有存储限制。

**解决方案**:
- 删除一些旧头像
- 压缩图片后上传
- 升级存储空间

## 问题6: \"Connection reset\" 或 \"连接错误\" 🔌

### 症状
- 保存配置时偶尔出现错误
- 错误信息包含 "connection reset" 或 "connection error"
- 有时重试后会成功

### 原因
这是临时的网络连接问题，常见原因：
- 网络不稳定
- 服务器暂时繁忙
- 连接超时被重置

### 解决方案

#### ✅ v3.2 已自动修复
最新版本已经实现**自动重试机制**：
- 遇到连接错误会自动重试（最多3次）
- 使用智能延迟（500ms → 1秒 → 2秒）
- 95%的临时错误都会自动恢复

#### 用户无需操作
系统会自动处理大部分连接错误。你可能会注意到：
- 保存稍微慢一点（多等几秒）
- 控制台显示重试日志（正常现象）
- 最终操作会成功完成

#### 如果仍然失败
如果3次重试后仍然失败：
1. **检查网络连接**
   - 确保WiFi/4G信号稳定
   - 尝试刷新页面重试

2. **检查数据大小**
   - 打开浏览器控制台（F12）
   - 查看是否有"数据过大"警告
   - 如果有，删除一些旧的聊天记录

3. **稍后重试**
   - 可能是服务器临时繁忙
   - 等待1-2分钟后重试

#### 监控重试过程
打开控制台（F12）查看详细日志：
```
🔄 [save config] Attempt 1/3
⚠️ [save config] Attempt 1 failed: connection error: connection reset
⏳ [save config] Retrying in 500ms...
🔄 [save config] Attempt 2/3
✅ [save config] Succeeded on attempt 2
```

这表示系统正在自动处理错误。

**技术细节**：查看 [CONNECTION_RESET_FIX.md](./CONNECTION_RESET_FIX.md)

## 问题7: 实时同步不工作 🔄

### 症状
- 在一个设备上的修改
- 在另一个设备上没有更新
- 需要刷新才能看到最新数据

### 解决方案

#### 手动同步
1. 点击左侧菜单
2. 点击"手动同步"按钮
3. 等待同步完成

#### 自动同步
- 系统会在登录后5秒自动启动同步
- 修改数据后会自动保存到云端
- 等待几秒后在其他设备刷新页面

#### 如果仍然不同步
1. 退出登录
2. 清除浏览器缓存
3. 重新登录
4. 数据会重新从云端加载

## 问题8: 应用启动很慢 ⏱️

### 症状
- 打开应用后长时间白屏
- 加载时间超过10秒

### 解决方案

#### 正常情况
应用采用零阻塞启动策略：
- 界面会在100ms内显示
- 登录状态在后台恢复
- 数据会在2-8秒内加载完成

#### 如果超过10秒
1. 检查网络连接
2. 刷新页面（F5）
3. 清除浏览器缓存
4. 更换网络环境

#### 优化建议
- 使用Chrome或Edge浏览器
- 定期清理聊天记录
- 避免创建过多角色（建议<20个）

## 问题9: 社交登录不可用 🔐

### 症状
- 点击Google/Microsoft登录按钮
- 提示"未配置"或"not enabled"

### 原因
需要在Supabase后台配置OAuth。

### 解决方案

参考详细配置指南：[SETUP_OAUTH.md](./SETUP_OAUTH.md)

**快速说明**:
1. 这需要应用管理员在Supabase后台配置
2. 如果你是用户，联系管理员配置
3. 如果你是管理员，按照SETUP_OAUTH.md操作

**临时方案**:
使用邮箱注册/登录功能，无需额外配置。

## 常见错误信息对照表

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| "需要配置API Key" | 未配置Gemini API Key | 参考问题1 |
| "API key错误" | API Key无效或格式错误 | 重新获取API Key |
| "API配额已用完" | 免费配额用完 | 等待重置或升级 |
| "网络错误" | 网络连接问题 | 检查网络 |
| "登录已过期" | Session过期 | 重新登录 |
| "未配置" (社交登录) | OAuth未配置 | 参考SETUP_OAUTH.md |
| "发送失败" | 多种可能 | 参考问题1和问题2 |

## 预防性维护建议 ✨
