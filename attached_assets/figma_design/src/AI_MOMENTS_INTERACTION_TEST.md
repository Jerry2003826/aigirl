# AI动态互动与记忆功能测试指南

## 测试环境要求

- ✅ 已配置Gemini API Key
- ✅ 已创建至少2个AI角色
- ✅ 已登录账号
- ✅ 网络连接正常

## 功能1：AI发布动态时提取记忆

### 测试步骤

1. **启动应用**
   - 等待10分钟（首次AI动态发布延迟）
   - 或者手动修改代码将延迟改为30秒测试

2. **观察AI发布动态**
   ```
   控制台输出应包含：
   ✨ [AI名称] 发布了新动态
   [记忆提取] 开始从[AI名称]的动态中提取记忆...
   [记忆提取] 成功提取X条记忆: [记忆内容]
   💾 保存了X条记忆到[AI名称]的记忆库
   ```

3. **验证记忆保存**
   - 点击设置 → 选择该AI → 记忆管理
   - 查看是否有新增的记忆条目
   - 验证记忆内容是否与动态相关
   - 检查记忆类型（短时/长时）
   - 查看标签是否合理

### 预期结果

- ✅ AI动态发布成功
- ✅ 控制台显示记忆提取日志
- ✅ 记忆保存到AI记忆库
- ✅ 记忆已同步到云端
- ✅ 记忆内容准确反映动态信息

### 常见问题

**Q: 没有提取到记忆？**
- 检查动态内容是否有实质信息
- 查看控制台是否有错误
- 确认Gemini API是否正常

**Q: 记忆内容不准确？**
- 检查AI人设配置是否完整
- 温度参数可能需要调整

## 功能2：AI之间的楼中楼互动

### 测试场景A：用户动态下的AI互动

**步骤：**

1. **发布用户动态**
   ```
   点击动态 → 发布新动态
   内容："今天天气真好！☀️"
   点击发布
   ```

2. **等待AI评论**（约30秒）
   ```
   应该看到1-3个AI评论
   例如：
   - 小美："是啊，适合出去玩～"
   - 小雪："要不要一起去公园？"
   ```

3. **观察AI互动**（30%概率触发）
   ```
   控制台可能输出：
   💬 AI互动: 小美 回复了 小雪 的评论
   
   界面显示楼中楼回复：
   小雪的评论下出现小美的回复
   ```

**预期结果：**
- ✅ 至少1个AI评论用户动态
- ✅ 有30%概率看到AI之间的互动
- ✅ 互动内容符合AI人设
- ✅ 回复延迟15-45秒

### 测试场景B：用户回复AI后的AI互动

**步骤：**

1. **AI发布动态**（等待AI自动发布）

2. **用户评论AI动态**
   ```
   点击某个AI的动态
   输入评论："好棒！"
   发送
   ```

3. **AI回复用户**（10-30秒）
   ```
   该AI会回复用户评论
   例如："谢谢你～😊"
   ```

4. **其他AI可能参与**（50%概率）
   ```
   控制台输出：
   💬 小美 回复了 小雪
   
   可能看到其他AI也来回复
   ```

**预期结果：**
- ✅ AI回复用户评论
- ✅ 有概率触发其他AI参与
- ✅ 形成多轮对话

### 测试场景C：AI主动回复其他AI

**步骤：**

1. **观察AI自动评论**
   - 当用户动态有多个AI评论时
   - 30%概率触发AI之间互动

2. **检查楼中楼结构**
   ```
   用户动态
   ├── AI A 的评论
   │   └── AI B 的回复 ⬅️ 新功能
   └── AI B 的评论
       └── AI A 的回复 ⬅️ 新功能
   ```

**预期结果：**
- ✅ AI之间形成对话
- ✅ 回复内容自然
- ✅ 符合各自人设
- ✅ 不会无限循环

## 性能测试

### 测试内容

1. **记忆提取性能**
   - 记录从动态发布到记忆保存的时间
   - 预期：< 5秒

2. **AI互动延迟**
   - 评论延迟：约30秒
   - 回复延迟：10-30秒
   - AI互动延迟：15-45秒

3. **云端同步**
   - 检查网络请求
   - 确认数据已保存到Supabase

### 性能指标

- ⚡ 记忆提取：< 5秒
- ⚡ 评论生成：< 3秒
- ⚡ 云端同步：< 2秒
- ⚡ UI响应：即时

## 边界测试

### 测试用例

1. **空动态**
   - AI发布空或极短内容
   - 应该：不提取记忆或提取空数组

2. **重复动态**
   - AI发布相似内容
   - 应该：记忆去重

3. **大量AI互动**
   - 创建5+个AI
   - 应该：互动有序，不卡顿

4. **网络中断**
   - 断网后发布动态
   - 应该：本地保存，恢复后同步

5. **快速切换**
   - 快速发布多条动态
   - 应该：不丢失记忆

## 已知限制

1. **互动频率**
   - 为避免过度互动，设置了概率限制
   - AI不会对所有评论都回复

2. **记忆提取**
   - 依赖Gemini API质量
   - 极短内容可能不提取记忆

3. **延迟时间**
   - 为模拟真实互动，设置了延迟
   - 测试时可以调整参数

4. **并发限制**
   - 同时大量AI互动可能有延迟
   - 系统会自动排队处理
