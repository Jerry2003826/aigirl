# AI女友聊天应用 - 完整用户指南

一个功能完整的AI女友聊天应用，支持多角色管理、群聊、自定义头像、图片识别、智能记忆和跨设备同步。

---

## 🚀 快速开始

### 第一步：登录/注册

应用支持三种登录方式：
1. **邮箱注册/登录**：填写邮箱和密码
2. **Google账号登录**：点击Google图标
3. **Microsoft账号登录**：点击Microsoft图标

> ⚠️ **社交登录配置**: Google和Microsoft登录需要在Supabase后台配置OAuth才能使用。未配置时会提示错误信息。详见下方"社交登录配置"章节。

### 第二步：配置API

1. 点击右上角"⚙️设置"
2. 在"API配置"中填入**Gemini API Key**（推荐）
3. 或配置其他兼容OpenAI格式的API

**获取Gemini API Key**: 访问 [Google AI Studio](https://aistudio.google.com/apikey)

**推荐模型**: `gemini-2.5-pro` 或 `gemini-2.0-flash-exp`

### 第三步：创建AI女友

1. 进入"设置" → "角色管理"
2. 点击"新建角色"
3. 填写基本信息：
   - 姓名
   - 年龄
   - 角色（女友、闺蜜、姐姐等）
4. 上传自定义头像（JPG/PNG）
5. 编辑性格、外貌、经历等详细设定
6. 保存

### 第四步：开始聊天

1. 在聊天列表中选择AI女友
2. 输入消息开始对话
3. 可以发送图片进行视觉交互
4. 系统会自动保存聊天记忆

## ✨ 核心功能

### 1. 多AI角色管理 👥

- 创建无限个不同性格的AI角色
- 每个角色独立的聊天记录和记忆
- 详细的Prompt管理系统
- 支持自定义头像、性格、背景故事

### 2. AI群聊功能 🎭

#### 创建群聊
1. 点击底部"群聊"标签
2. 点击"创建群聊"
3. 输入群聊名称
4. 创建完成（默认是空群）

#### 添加成员
1. 进入群聊
2. 点击右上角"人员"图标
3. 勾选要添加的AI
4. 点击"添加成员"

#### 群聊特性
- **智能回复选择**: 系统自动选择最相关的1-2个AI回复
- **@提及功能**: 使用 `@AI名字` 指定特定AI回复
- **防止重复**: 智能相似度检测，避免重复内容
- **追问机制**: AI可以智能追问澄清信息（有限制）
- **话题管理**: 每个话题最多10轮对话，自动总结
- **按拼音排序**: 群聊列表按首字母排序

#### 群聊设置
- 话题最大轮数：5-20轮（默认10）
- 冷却时间：1000-5000ms（默认2000）
- 追问次数上限：1-5次（默认3）
- 消息最大长度：50-300字（默认150）

### 3. 智能记忆系统 🧠

- **自动记忆提取**: AI会从对话中提取重要信息
- **记忆分类**: 
  - 基本信息（姓名、生日、职业等）
  - 喜好偏好（喜欢的食物、颜色等）
  - 重要事件（纪念日、经历等）
  - 关系状态
- **去重机制**: 自动合并相似记忆
- **记忆查看**: 在聊天界面右上角查看所有记忆

### 4. Gemini AI引擎 🤖

只需一个Gemini API Key即可使用：
- ✅ 文字对话
- ✅ 图片识别（Vision）
- ✅ RAG语义检索
- ✅ 联网搜索（Google Search Grounding）

### 5. 跨设备同步 ☁️

- 所有数据保存在Supabase云端
- 支持多设备自动同步
- 聊天记录、角色设定、记忆全部云端存储
- 实时同步更新

### 6. 自定义头像 🖼️

**上传方法**:
1. 进入"设置" → "角色管理"
2. 选择要编辑的角色
3. 在"基本信息"卡片顶部点击"上传头像"
4. 选择图片文件（JPG/PNG，建议小于1MB）
5. 保存

头像会自动上传到Supabase Storage并在所有设备同步。

## 🔐 社交登录配置

### Google登录配置

1. 访问 [Google Cloud Console](https://console.cloud.google.com)
2. 创建OAuth 2.0凭据
3. 获取Client ID和Client Secret
4. 在Supabase Dashboard中配置：
   - 进入 **Authentication** > **Providers** > **Google**
   - 填入Client ID和Secret
   - 设置回调URL: `https://[你的项目ID].supabase.co/auth/v1/callback`
5. 保存并启用

详细步骤：[Supabase Google OAuth文档](https://supabase.com/docs/guides/auth/social-login/auth-google)

### Microsoft登录配置

1. 访问 [Azure Portal](https://portal.azure.com)
2. 创建应用注册
3. 获取Client ID和Client Secret
4. 在Supabase Dashboard中配置：
   - 进入 **Authentication** > **Providers** > **Azure**
   - 填入Client ID和Secret
   - 设置重定向URI: `https://[你的项目ID].supabase.co/auth/v1/callback`
5. 保存并启用

详细步骤：[Supabase Azure OAuth文档](https://supabase.com/docs/guides/auth/social-login/auth-azure)

### 登录流程

1. 用户点击社交登录按钮
2. 跳转到OAuth授权页面
3. 用户授权后自动跳转回应用
4. 应用完成登录并加载数据

> 💡 **提示**: 首次社交登录会自动创建新账号，昵称从社交账号获取。

## ❓ 常见问题

### 功能类

**Q: 在哪里上传头像？**  
A: 设置 → 角色管理 → 选择角色 → 基本信息 → 上传头像按钮

**Q: 头像没有显示？**  
A: 确保点击了"保存"按钮。头像需要保存后才会在聊天界面显示。

**Q: 如何让群聊中特定AI回复？**  
A: 在消息中使用 `@AI名字`，例如：`@小雪 你觉得呢？`

**Q: 群聊人数有限制吗？**  
A: 技术上无限制，但建议每个群聊3-5个AI，获得最佳体验。

**Q: 可以创建多少个角色/群聊？**  
A: 没有数量限制，存储在云端Supabase中。

**Q: AI回复太频繁/太少？**  
A: 在群聊设置中调整"冷却时间"和"追问次数上限"。

### 登录类

**Q: 社交登录显示"未配置"？**  
A: 需要在Supabase后台配置OAuth。详见上方"社交登录配置"章节。

**Q: 登录后数据丢失？**  
A: 社交登录和邮箱登录是独立账号。确保使用相同的登录方式。

**Q: 忘记密码怎么办？**  
A: 目前暂不支持密码重置，请使用社交登录或联系管理员。

### 技术类

**Q: 数据存在哪里？**  
A: 所有数据（聊天记录、角色、记忆、头像）都保存在Supabase云端，支持跨设备同步。

**Q: API Key安全吗？**  
A: API Key保存在Supabase云端，采用安全的加密存储，不会泄露。

**Q: 是否支持离线使用？**  
A: 不支持。应用需要联网调用AI API和同步数据。

**Q: 删除的数据能恢复吗？**  
A: 删除操作不可撤销。建议重要数据定期导出备份。

## 🛠️ 故障排除

### AI不回复
- 检查是否设置了API Key
- 确认API Key有效且有配额
- 查看浏览器控制台是否有错误
- 确认网络连接正常

### 头像上传失败
- 检查图片格式（仅支持JPG/PNG）
- 确保图片大小不超过5MB
- 确认网络连接稳定
- 查看控制台错误日志

### 数据不同步
- 检查网络连接
- 刷新页面重新加载
- 确认使用相同账号登录
- 查看实时同步状态指示器

### 群聊AI重复回复
- 系统有相似度检测（70%阈值）
- 如果仍然重复，可增加冷却时间
- 调整话题轮换频率

### 登录超时
- 应用采用零阻塞启动策略
- 登录状态由后台自动恢复
- 如果长时间未恢复，请刷新页面
