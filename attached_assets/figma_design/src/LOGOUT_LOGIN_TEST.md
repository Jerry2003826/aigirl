# 登出-登录功能测试清单

## 🎯 测试目的
验证登出后再次登录不会出现无限加载问题。

---

## ✅ 测试步骤

### 测试1: 邮箱登录 → 登出 → 再次登录

1. **首次登录**
   - [ ] 使用邮箱和密码登录
   - [ ] 确认成功进入聊天界面
   - [ ] 发送1-2条测试消息

2. **登出**
   - [ ] 点击右上角设置（或个人资料）
   - [ ] 点击"登出"按钮
   - [ ] 看到"已登出"或"已退出登录"提示
   - [ ] 确认返回到登录页面

3. **再次登录**
   - [ ] 使用相同的邮箱和密码登录
   - [ ] **预期**: 1-3秒内进入聊天界面
   - [ ] **预期**: 看到之前的聊天记录
   - [ ] **预期**: 没有无限加载状态

4. **验证数据恢复**
   - [ ] 检查AI女友角色是否正常显示
   - [ ] 检查聊天记录是否完整
   - [ ] 尝试发送新消息，确认功能正常

### 测试2: Google OAuth登录 → 登出 → 再次登录

1. **首次Google登录**
   - [ ] 点击Google登录按钮
   - [ ] 完成Google授权
   - [ ] 确认成功进入聊天界面

2. **登出**
   - [ ] 点击登出按钮
   - [ ] 看到"已登出"提示
   - [ ] 返回登录页面

3. **再次Google登录**
   - [ ] 再次点击Google登录按钮
   - [ ] **预期**: 快速完成授权（可能跳过授权页面）
   - [ ] **预期**: 1-3秒内进入聊天界面
   - [ ] **预期**: 没有无限加载

### 测试3: 快速登出-登录循环测试

1. **压力测试**
   - [ ] 登录 → 立即登出
   - [ ] 再次登录 → 立即登出
   - [ ] 重复3-5次
   - [ ] **预期**: 每次都能正常登录/登出
   - [ ] **预期**: 没有卡住或无限加载

---

## 🔍 检查点

### 控制台输出（正常情况）

#### 登出时应该看到：
```
👋 用户已登出，清理所有状态...
🔄 恢复默认数据...
```

#### 登录时应该看到：
```
✅ 用户已登录: { userId: '...', email: '...', provider: '...' }
📥 开始加载用户数据...
✅ 用户数据加载成功
✅ [loadUserDataFromCloud] 数据加载成功，设置 isLoadingData = false
```

#### OAuth回调时应该看到：
```
🔐 检测到OAuth回调，code: xxxxxx...
🔄 正在用授权码换取session...
✅ 成功换取session: { userId: '...', email: '...' }
```

### 异常情况标志 ❌

如果看到以下情况，说明有问题：
- 页面一直显示加载动画（超过5秒）
- 控制台没有"设置 isLoadingData = false"日志
- 控制台显示错误信息
- URL一直显示`?code=xxx`（OAuth登录）

## 🐛 如遇问题

### 立即尝试
1. 打开浏览器控制台（F12）
2. 查看是否有红色错误信息
3. 截图控制台输出
4. 刷新页面重试

### 清除缓存
1. Chrome: `Cmd/Ctrl + Shift + Delete`
2. 选择"所有时间"
3. 勾选"Cookie"和"缓存"
4. 清除数据后重启浏览器

### 检查网络
- 确保Supabase服务可访问
- 检查是否有防火墙/代理阻拦
- 尝试切换网络（如切换到手机热点）

## ✨ 修复详情

此问题在2024-11-03已修复，主要改进：

1. **SIGNED_OUT事件处理**
   - 新增：重置`isLoadingData = false`
   - 新增：恢复默认数据状态
   - 新增：完整清理认证状态

2. **数据加载错误处理**
   - 新增：超时/失败时设置`setIsLoadingData(false)`
   - 改进：所有路径都正确处理loading状态

3. **OAuth回调处理**
   - 新增：`handleOAuthCallback`自动处理code交换
   - 新增：URL参数自动清理
