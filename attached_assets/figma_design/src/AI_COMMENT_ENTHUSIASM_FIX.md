# AI评论积极性优化

## 🎯 优化目标

**用户诉求：** 发动态后，希望AI能够快速、积极地回复评论

**之前的问题：**
- ❌ AI评论延迟太长（30秒）
- ❌ AI评论兴趣度低（50-70%）
- ❌ 参与评论的AI数量少（1-3个）

**现在的优化：**
- ✅ AI评论延迟缩短（10-30秒）
- ✅ AI评论兴趣度提高（80-95%）
- ✅ 参与评论的AI数量增加（至少2个，最多全部）

## 📊 详细改进

### 1. 评论延迟优化

**之前：**
```typescript
const delayMs = 30000; // 固定30秒
```

**现在：**
```typescript
const delayMs = 10000 + Math.random() * 20000; // 10-30秒随机
```

**效果：**
- ⚡ 最快10秒就会有AI评论
- ⚡ 平均20秒左右
- ⚡ 感觉更加自然和积极

### 2. 参与评论的AI数量优化

**之前：**
```typescript
const numCommenters = Math.floor(Math.random() * 3) + 1; // 1-3个AI
```

**现在：**
```typescript
const minCommenters = Math.min(2, this.personalities.length);
const maxCommenters = this.personalities.length;
const numCommenters = minCommenters + Math.floor(Math.random() * (maxCommenters - minCommenters + 1));
```

**效果：**
- 📈 至少2个AI会评论（如果有2个以上AI）
- 📈 最多所有AI都会评论
- 📈 平均会有大部分AI评论

**示例：**
- 2个AI：2个都会评论
- 3个AI：2-3个会评论
- 5个AI：2-5个会评论
- 10个AI：2-10个会评论

### 3. AI兴趣度优化

#### 之前的兴趣判断逻辑

```typescript
// 内容短：30%感兴趣
if (moment.content.length < 10) {
  return Math.random() > 0.7;
}

// 有图片：70%感兴趣
if (moment.images.length > 0) {
  return Math.random() > 0.3;
}

// 默认：50%感兴趣
return Math.random() > 0.5;
```

#### 现在的兴趣判断逻辑

```typescript
// 有图片：95%感兴趣 ⬆️
if (moment.images.length > 0) {
  return Math.random() > 0.05;
}

// 内容长：90%感兴趣 ⬆️
if (moment.content.length > 20) {
  return Math.random() > 0.1;
}

// 内容短：80%感兴趣 ⬆️
if (moment.content.length < 10) {
  return Math.random() > 0.2;
}

// 默认：85%感兴趣 ⬆️
return Math.random() > 0.15;
```

**改进对比：**

| 动态类型 | 之前兴趣度 | 现在兴趣度 | 提升 |
|---------|-----------|-----------|------|
| 有图片   | 70%       | 95%       | +25% |
| 内容长   | 50%       | 90%       | +40% |
| 内容短   | 30%       | 80%       | +50% |
| 普通     | 50%       | 85%       | +35% |

### 4. 评论间隔优化

**之前：**
```typescript
await new Promise(resolve => setTimeout(resolve, 2000 + Math.random() * 3000));
// 2-5秒间隔
```

**现在：**
```typescript
await new Promise(resolve => setTimeout(resolve, 1000 + Math.random() * 2000));
// 1-3秒间隔
```

**效果：**
- ⚡ 多个AI评论会更快地连续出现
- ⚡ 更有"热闹"的感觉

### 5. 防止重复调用优化

**问题：** 之前可能会重复安排AI评论

**解决方案：** 立即标记`commentScheduled = true`

**之前：**
```typescript
const newMoment: Moment = {
  // ...
  commentScheduled: false, // 初始为false
};
setMoments([newMoment, ...prevMoments]);

// 安排评论
scheduleAIComments(newMoment, (momentId, comment) => {
  // 在回调中才设置为true
  { ...m, commentScheduled: true }
});
```

**现在：**
```typescript
const newMoment: Moment = {
  // ...
  commentScheduled: true, // 立即标记为true
};
setMoments([newMoment, ...prevMoments]);

// 安排评论
scheduleAIComments(newMoment, (momentId, comment) => {
  // 不再重复设置
});
```

## 📊 性能影响

### API调用

**之前：** 平均每条动态调用1-3次Gemini API
**现在：** 平均每条动态调用2-所有AI次Gemini API

**示例（5个AI）：**
- 之前：平均2次API调用
- 现在：平均3-4次API调用
- 增加：约50-100%

### 响应时间

- 单次评论生成：1-3秒
- 多个AI依次评论：5-15秒
- 不会阻塞主线程（异步执行）

### 成本考虑

如果使用Gemini API：
- 免费配额：每分钟15次请求
- 发布1条动态可能触发3-5次评论
- 建议：每分钟不要发布超过3条动态
