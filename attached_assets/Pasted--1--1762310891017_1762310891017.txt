大概可以这样理解：你给每个会话单独记了一条“未读计数”，谁在你没打开的情况下收到了新消息，这个数字就涨；只要你真正点进这个对话，它就被清零。

我帮你拆成几块说，方便你写文档 👇

1. 未读数是怎么存的？

在你的状态里，每个聊天（女友一对一 / 群聊）都有一份自己的数据结构，里面除了 messages 之外，还挂了一个 unreadCount（未读条数） 和最近一条消息时间：

type Chat = {
  personalityId: string;
  messages: Message[];
  unreadCount: number;        // 当前会话未读条数
  lastMessageAt?: number;     // 用来排序和显示时间
};


这些东西都放在 App.tsx 维护的 chats 里，
比如 chats[personalityId].unreadCount 就是这位女友的未读数。

整个 chats 会随着云端同步一起保存 / 载入，所以未读数也会跨设备同步。

2. 什么情况下“未读数 +1”？

核心就是：只要会话收到新消息，但你当前不在这个会话里看，它就算“未读”。

在 App.tsx 里，你有一个类似这样的更新函数（概念上）：

function updateChatMessages(
  personalityId: string,
  newMessages: Message[],
  markAsRead?: boolean  // 重点：这个标志控制“要不要清零未读”
) {
  setChats(prev => {
    const prevChat = prev[personalityId] ?? createEmptyChat(personalityId);
    const oldLen = prevChat.messages.length;
    const newLen = newMessages.length;
    const hasNew = newLen > oldLen;

    let unread = prevChat.unreadCount;

    if (markAsRead) {
      // 标记为已读：未读数直接清零
      unread = 0;
    } else if (hasNew) {
      // 有新消息，而且没标记为已读：把“新增加的那几条”加到未读里
      unread += newLen - oldLen;
    }

    return {
      ...prev,
      [personalityId]: {
        ...prevChat,
        messages: newMessages,
        unreadCount: unread,
        lastMessageAt: newMessages[newLen - 1]?.timestamp ?? prevChat.lastMessageAt,
      },
    };
  });
}


所以：

当别的地方（比如云端同步 / 群聊、AI 自动发消息）给某个会话 append 新消息时，如果没有传 markAsRead = true，未读数就会 按新增消息数量往上加。

你不在那个对话里看的时候，这些全被当成“未读”。

3. 什么时候未读数会被清零？

关键点在 ChatInterface 里你传的那个第二个参数：

setMessages(newMessages, true); // 第二个参数 true = 把这个会话标记成“已读”


你在几个地方是这么用的：

你打开会话时
App 把当前女友的 messages 传给 ChatInterface，
在初始化 / 切换对话时，会调用一次 setMessages(messages, true)，
→ 这个对话所有未读立刻被当成“已读”，unreadCount = 0。

你在这个对话里主动发消息时
sendMessage 里先构造 userMessage，然后：

const newMessages = [...messages, userMessage];
setMessages(newMessages, true);


表示：这条消息及之前的都在你自己眼皮底下发生，所以不算未读，直接清空未读数。

你正在看这个会话时，对方（AI 女友）回消息
生成完 assistantMessage 后，同样走 setMessages([...newMessages, assistantMessage], true)，
因为你当前就在这个聊天窗口里，这些新消息自然“即时已读”。

反过来，如果是你没打开的会话：

例如你在 A 女友聊天界面，这时候 B 女友那边有云端同步推过来一条新消息，App 会更新 B 会话的 messages，但不会带 markAsRead = true；

于是 B 的 unreadCount 就会被 +1，这就是“你有一条来自 B 的未读”。

4. ChatList 上是怎么显示红点 / 未读小圆角的？

在 ChatList 渲染每一行会话的时候，会从 chat.unreadCount 里读数字：

{chat.unreadCount > 0 && (
  <div className="ml-2 px-2 min-w-[1.5rem] h-5 rounded-full bg-red-500 text-xs text-white flex items-center justify-center">
    {chat.unreadCount > 99 ? '99+' : chat.unreadCount}
  </div>
)}


逻辑是：

unreadCount === 0：不渲染角标，看起来就是“没有未读”。

unreadCount > 0：右侧出现一个红色圆角角标：

宽度至少 1.5rem，高度 20px 左右；

居中显示数字；

数字大于 99 的时候显示为 99+，避免太宽。

手机 / 电脑这块逻辑是一样的，只是 ChatList 在手机上会单独占满屏，电脑上是在左侧栏。
